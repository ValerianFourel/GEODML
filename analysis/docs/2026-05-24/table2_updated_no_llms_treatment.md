## Methodology note for the paper

The previous version of Table 2 included $T_\ell\!=\!\texttt{has\_llms\_txt}$
as a 7th treatment, with reported coefficients
$+0.007^{**}$ / $+0.076^{**}$ / $-0.019$ on the three outcomes.
We **remove** that row from Table 2 because the LLM rerankers used in
this study (Llama-3.3-70B-Instruct and Qwen2.5-72B-Instruct) **do not
retrieve or read the \texttt{/llms.txt} file at inference time**: the
file's content never enters the model's context window, so any
"effect" the marginal estimator picked up was a confounded
correlation with other site-level signals, not a causal pathway from
\texttt{has\_llms\_txt} to ranking.

We instead include \texttt{has\_llms\_txt} in the confounder set $X$
as a **proxy for unobserved site-level GEO intent**. Sites that
publish \texttt{/llms.txt} have signaled willingness to invest in LLM
optimization, and that latent intent plausibly correlates with the
other content treatments ($r = +0.20$ with $T_3$ schema; $r = +0.18$
with $T_6$ freshness; $|r|\!\leq\!0.20$ overall, see
Appendix \ref{app:llms-correlations}). Including
\texttt{has\_llms\_txt} in $X$ controls for that latent factor so
that the effects we estimate for $T_1$–$T_6$ are not contaminated by
it. The coefficient on \texttt{has\_llms\_txt} itself is not reported
because it is a nuisance variable under this framing.

The headline coefficients are essentially unchanged from the previous
specification: all six remaining treatments move by $\leq 0.022$ in
either direction, and no significance category changes. The
$T_\ell$ row is the only structural change.
